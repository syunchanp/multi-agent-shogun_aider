## 1. 要件定義

### 1.1 プロジェクト概要

TravelPlannerは、旅行計画の立案から記録までを一元管理するWebサービスである。

**ミッション**: 旅行者のコアプラットフォームとして、計画・予約・記録のすべてを統合する。

### 1.2 機能要件

#### コア機能

| ID | 機能名 | 説明 | 優先度 |
|----|--------|------|--------|
| F001 | 旅行計画作成 | 旅行の日程・行程を作成・管理 | 必須 |
| F002 | 移動計画管理 | 交通手段（電車・飛行機・バス等）の計画 | 必須 |
| F003 | 宿泊計画管理 | 宿泊施設の予約情報管理 | 必須 |
| F004 | 外部リンク管理 | 交通系サービス（JR予約、航空会社等）へのリンク | 必須 |
| F005 | 写真紐付け | Google Photos連携による旅行写真の紐付け | 必須 |
| F006 | 地図表示 | 行程・写真を地図上にプロット | 高 |
| F007 | タイムライン表示 | 旅行の時系列表示 | 高 |
| F008 | 共有機能 | 旅行計画を他ユーザーと共有 | 中 |
| F009 | エクスポート | 計画をPDF/カレンダー形式で出力 | 中 |
| F010 | テンプレート | 人気ルートのテンプレート提供 | 低 |

#### Google Photos連携（制約あり）

**重要**: 2025年3月のAPI変更により、以下の制約がある。

| 機能 | 実現可否 | 実装方式 |
|------|---------|---------|
| ユーザーの既存写真を自動取得 | ❌ 不可 | - |
| ユーザーに写真を選択させる | ✅ 可 | Picker API |
| 選択写真のGPS情報取得 | ✅ 可 | Picker API経由 |
| 選択写真のEXIF取得 | ✅ 可（制限あり） | Picker API経由 |
| アプリ内で撮影した写真の管理 | ✅ 可 | Library API |

**設計方針**:
- Picker APIを使用し、ユーザーがGoogle提供のUIで写真を選択する方式を採用
- 選択された写真のメタデータ（GPS、撮影日時）を取得し、旅行記録に紐付け

### 1.3 ユースケース

```
UC1: 旅行計画を作成する
  アクター: ユーザー
  1. ユーザーが「新規旅行」を作成
  2. 旅行名・日程を入力
  3. 行程（目的地）を追加
  4. 移動手段・宿泊先を登録
  5. 計画を保存

UC2: 旅行写真を紐付ける
  アクター: ユーザー
  1. ユーザーが旅行詳細画面を開く
  2. 「写真を追加」ボタンをクリック
  3. Google Photos Picker UIが表示
  4. ユーザーが写真を選択
  5. システムがGPS・撮影日時を抽出
  6. 写真が地図・タイムラインに表示

UC3: 旅行を振り返る
  アクター: ユーザー
  1. 過去の旅行を選択
  2. タイムライン形式で行程を確認
  3. 地図上で移動ルートと写真を確認
  4. 思い出を共有（オプション）
```

### 1.4 ユーザーペルソナ

| ペルソナ | 特徴 | ニーズ |
|---------|------|--------|
| 計画好き旅行者 | 事前に詳細な計画を立てる | 一元管理、リンク整理 |
| 写真記録派 | 旅行中に大量の写真を撮影 | 自動整理、地図プロット |
| カジュアル旅行者 | ざっくり計画、柔軟に行動 | シンプルUI、テンプレート |

### 2.4 API設計（主要エンドポイント）

```yaml
# 認証
POST   /api/auth/login          # Googleログイン開始
GET    /api/auth/callback       # OAuth コールバック
POST   /api/auth/refresh        # トークンリフレッシュ
POST   /api/auth/logout         # ログアウト

# 旅行
GET    /api/trips               # 旅行一覧
POST   /api/trips               # 旅行作成
GET    /api/trips/:id           # 旅行詳細
PUT    /api/trips/:id           # 旅行更新
DELETE /api/trips/:id           # 旅行削除

# 行程
GET    /api/trips/:id/itinerary          # 行程一覧
POST   /api/trips/:id/itinerary          # 行程追加
PUT    /api/trips/:id/itinerary/:dayId   # 行程更新

# アクティビティ
POST   /api/itinerary/:dayId/activities        # アクティビティ追加
PUT    /api/activities/:id                     # アクティビティ更新
DELETE /api/activities/:id                     # アクティビティ削除

# 写真（Google Photos連携）
POST   /api/trips/:id/photos/picker-session   # Picker セッション開始
GET    /api/trips/:id/photos                  # 紐付け写真一覧
POST   /api/trips/:id/photos                  # 写真紐付け
DELETE /api/trips/:id/photos/:photoId         # 写真紐付け解除

# 外部リンク
GET    /api/trips/:id/links     # リンク一覧
POST   /api/trips/:id/links     # リンク追加
DELETE /api/links/:id           # リンク削除

# 共有
POST   /api/trips/:id/share     # 共有設定
GET    /api/shared/:token       # 共有リンクアクセス
```

### 3.2 構成詳細

| コンポーネント | サービス | スペック | 理由 |
|--------------|---------|---------|------|
| CDN | CloudFront | グローバル | グローバル配信、SSL終端 |
| WAF | AWS WAF | マネージドルール | SQL injection、XSS防止 |
| ロードバランサー | ALB | 2AZ | トラフィック分散 |
| コンピュート | ECS Fargate | Graviton (ARM) | コールドスタートなし、コスト効率 |
| データベース | Aurora Serverless v2 | 0.5-8 ACU | 自動スケーリング |
| キャッシュ | ElastiCache Redis | cache.t4g.micro | セッション管理（オプション） |
| ストレージ | S3 | Intelligent-Tiering | 写真キャッシュ |
| シークレット | Secrets Manager | - | OAuth トークン等 |
| DNS | Route 53 | - | ドメイン管理 |

## 5. 非機能要件

### 5.1 性能要件

| 指標 | 目標値 | 測定方法 |
|------|-------|---------|
| ページ読み込み（初回） | < 3秒 | Lighthouse |
| API レスポンス（95%ile） | < 500ms | CloudWatch |
| 同時接続ユーザー | 1,000 | 負荷テスト |
| 地図表示 | < 2秒 | 実測 |

### 5.2 可用性要件

| 項目 | 目標 | 構成 |
|------|------|------|
| 稼働率 | 99.9%（年間8.76時間以下のダウンタイム） | Multi-AZ |
| RTO（目標復旧時間） | 1時間 | Aurora自動フェイルオーバー |
| RPO（目標復旧地点） | 5分 | Aurora継続的バックアップ |

### 5.3 セキュリティ要件

#### 認証・認可

| 項目 | 実装 |
|------|------|
| 認証方式 | OAuth 2.0 (Google) |
| セッション管理 | JWT + Refresh Token |
| トークン保存 | HttpOnly Cookie + Secure |
| CSRF対策 | SameSite=Strict |

#### データ保護

| 項目 | 実装 |
|------|------|
| 通信暗号化 | TLS 1.3 |
| 保存時暗号化 | Aurora: AES-256, S3: SSE-S3 |
| 個人情報 | GDPR準拠のデータ取り扱い |
| シークレット | AWS Secrets Manager |

#### インフラセキュリティ

| 項目 | 実装 |
|------|------|
| ネットワーク | VPC Private Subnet + NAT Gateway |
| WAF | AWS Managed Rules |
| 脆弱性スキャン | Amazon Inspector |
| 監査ログ | CloudTrail |

### 5.4 スケーラビリティ

| コンポーネント | スケーリング方式 |
|--------------|----------------|
| ECS Fargate | Target Tracking Auto Scaling (CPU 70%) |
| Aurora Serverless v2 | 自動スケーリング (0.5-8 ACU) |
| CloudFront | グローバル自動 |

## 7. 開発ロードマップ

### 7.1 フェーズ概要

```
Phase 1: MVP        Phase 2: 成長       Phase 3: 拡張
(3ヶ月)             (3ヶ月)             (3ヶ月〜)
────────────────────────────────────────────────────→

[基本機能]          [写真連携強化]       [共有・コミュニティ]
[認証]              [地図機能強化]       [テンプレート]
[行程管理]          [エクスポート]       [AI推奨]
```

### 7.2 Phase 1: MVP（3ヶ月）

**目標**: 基本的な旅行計画・管理が可能な状態でローンチ

| Sprint | 期間 | 成果物 |
|--------|------|--------|
| Sprint 1-2 | 2週間 x2 | インフラ構築、認証基盤 |
| Sprint 3-4 | 2週間 x2 | 旅行CRUD、行程管理 |
| Sprint 5-6 | 2週間 x2 | 外部リンク管理、基本UI |

**MVP機能スコープ**:
- ✅ Googleログイン
- ✅ 旅行作成・編集・削除
- ✅ 行程（日程・目的地）管理
- ✅ 移動計画・宿泊計画の登録
- ✅ 外部リンク管理
- ⏳ 写真連携（Phase 2）

### 7.3 Phase 2: 成長（3ヶ月）

**目標**: 写真連携と地図機能で差別化

| Sprint | 期間 | 成果物 |
|--------|------|--------|
| Sprint 7-8 | 2週間 x2 | Google Photos Picker統合 |
| Sprint 9-10 | 2週間 x2 | 地図表示強化、ルート可視化 |
| Sprint 11-12 | 2週間 x2 | PDF/カレンダーエクスポート |

**Phase 2機能スコープ**:
- ✅ Google Photos連携（Picker API）
- ✅ 写真のGPS抽出・地図プロット
- ✅ タイムライン表示強化
- ✅ PDF/iCal エクスポート

### 7.4 Phase 3: 拡張（3ヶ月〜）

**目標**: コミュニティ機能とAI活用

| 機能 | 優先度 | 概要 |
|------|--------|------|

- ドキュメント更新
